# Deploying Agent-Aware

This app is packaged as a Docker container that runs anywhere that can run
Docker (Render, Railway, Fly.io, Google Cloud Run, a VPS, etc.). Secrets are
injected at runtime as environment variables — nothing sensitive is baked into
the image.

> **Resource note:** the app runs headless Chromium (Playwright) **and** a
> sentence-transformer embedding model. Give it **≥ 1 GB RAM** for reliable
> searches. Free 512 MB tiers will often OOM mid-search.

---

## What you must do yourself (I can't do these for you)

These are account/consent/billing actions:
1. Finish the **Google OAuth client** (below) and copy its Client ID + Secret.
2. Create an account on a **hosting platform** (Render is used as the example).
3. Paste your secret values into that platform and click **Deploy**.

Everything else (the container, startup, headless config, secret wiring) is
already built.

---

## Step 1 — Finish the Google OAuth client

You already created the Google Cloud project. Now:

1. **Google Auth Platform** (where you saw "not configured yet") → **Get started**
   - App name: `Agent-Aware`, pick your support email.
   - Audience: **External**.
   - Add your developer contact email → finish.
2. Left menu → **Audience** → under *Test users*, **Add** every Gmail address
   that should be able to sign in (including yours). *(Required while the app is
   in "Testing" status.)*
3. Left menu → **Clients** → **Create client**
   - Application type: **Web application**
   - **Authorized redirect URIs** → add **both** of these (you can add the
     production one now or after Step 2 once you know your URL):
     ```
     http://localhost:8501/oauth2callback
     https://YOUR-APP-URL/oauth2callback
     ```
   - **Create** → copy the **Client ID** and **Client Secret**.

---

## Step 2 — Deploy to Render (example)

1. Push this repo to GitHub (see "Pushing to GitHub" below).
2. In Render: **New → Blueprint** → connect the repo. Render reads `render.yaml`.
3. Render will prompt for the env vars marked `sync: false`. Fill in:

   | Variable | Value |
   |---|---|
   | `GROQ_API_KEY` | your Groq key |
   | `GEMINI_API_KEY` | (optional) Gemini key — splits LLM load with Groq + auto-failover |
   | `TAVILY_API_KEY` | your Tavily key |
   | `RAPIDAPI_KEY` / `RAPIDAPI_HOST` | (optional) your RapidAPI key + host |
   | `GOOGLE_CLIENT_ID` | from Step 1 |
   | `GOOGLE_CLIENT_SECRET` | from Step 1 |
   | `APP_URL` | your Render URL, e.g. `https://agent-aware.onrender.com` |
   | `ALLOWED_EMAILS` | (optional) `you@gmail.com,friend@gmail.com` |

   (`COOKIE_SECRET` is auto-generated by Render; `PLAYWRIGHT_HEADLESS=true` is preset.)
4. Click **Apply / Deploy**. First build takes a few minutes (it installs Chromium).

### Step 3 — Point Google at the live URL
Once Render gives you the final URL, make sure:
- `APP_URL` env var = that URL (redeploy if you changed it), **and**
- the same URL + `/oauth2callback` is in your Google client's redirect URIs.

Visit the URL → you should get the **Continue with Google** screen.

## Run the container locally first (recommended)

```bash
docker build -t agent-aware .

docker run -p 8501:8501 \
  -e GROQ_API_KEY=your_key \
  -e TAVILY_API_KEY=your_key \
  -e GOOGLE_CLIENT_ID=your_id \
  -e GOOGLE_CLIENT_SECRET=your_secret \
  -e APP_URL=http://localhost:8501 \
  -e ALLOWED_EMAILS=you@gmail.com \
  agent-aware
```

Open http://localhost:8501. Omit the `GOOGLE_*` vars to run in **open mode**
(no login) for a quick smoke test.

## Pushing to GitHub

`.gitignore` already excludes `.env`, `.streamlit/secrets.toml`, `data/`, and
logs, so your secrets won't be committed. Verify before the first push:

```bash
git status            # confirm .env and secrets.toml are NOT listed
git add .
git commit -m "Add Google auth gate + Docker deploy"
git remote add origin https://github.com/<you>/agent-aware.git
git push -u origin main
```

## Other platforms (quick notes)

- **Railway / Fly.io / Cloud Run**: all deploy this Dockerfile directly. Set the
  same env vars from the table above in their dashboards. They inject `$PORT`,
  which the entrypoint already respects.
- **Persistence (optional)**: `data/` (price history + reliability stats) is
  ephemeral by default and resets on redeploy. To keep it, attach a persistent
  disk mounted at `/app/data`.
- **Streamlit Community Cloud**: not recommended here — it can't run the
  Playwright browser automation this app uses.
